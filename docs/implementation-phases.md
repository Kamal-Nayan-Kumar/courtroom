# AI Courtroom Implementation Plan & Phases

This document outlines the step-by-step process for building the backend and AI agents for the AI Courtroom project. We will focus on iteration, starting from a solid foundation and gradually layering in the multi-agent complexities of a trial.

## Phase 1: Foundation & Infrastructure (Backend Setup)
*   **Goal:** Set up the project structure, API routes, and database connections.
*   **Tasks:**
    1.  Initialize Python environment using `uv`.
    2.  Set up FastAPI server with standard REST routes and WebSocket support.
    3.  Define the foundational Pydantic models for `Case`, `User`, `Transcript`, and `Evidence`.
    4.  Set up LLM API integrations (Gemini, Sarvam, Perplexity).
*   **Deliverable:** A running FastAPI server that can accept API keys and connect to a mock database/vector store.

## Phase 2: RAG Pipeline & Data Ingestion
*   **Goal:** Build the system that processes uploaded case files and pre-made cases.
*   **Tasks:**
    1.  Create endpoints for users to upload custom case PDFs/Text.
    2.  Implement Document Loaders (LangChain) and Text Splitters optimized for legal text (preserving IPC/BNS sections).
    3.  Set up Vector Database (e.g., Qdrant, Chroma).
    4.  Build a retrieval tool that agents can use to query evidence and case facts.
*   **Deliverable:** A functional RAG endpoint where querying "What was the murder weapon?" returns the exact excerpt from the uploaded case file.

## Phase 3: Single Agent Engineering (Prompts & Tools)
*   **Goal:** Build the individual "brains" of the Judge, Prosecutor, and Defender.
*   **Tasks:**
    1.  Design strict system prompts for the **AI Judge** (focusing on Indian Law, impartiality, and evidence ruling).
    2.  Design system prompts for the **Complainant's Counsel/Prosecutor** (aggressive, evidence-based).
    3.  Design system prompts for the **Defender** (fact-checking, finding loopholes).
    4.  Equip agents with the Perplexity Tool for web search and the RAG Tool for case facts.
*   **Deliverable:** Three separate LangChain agents that can answer questions independently given a set of facts.

## Phase 4: Multi-Agent Orchestration (LangGraph)
*   **Goal:** Connect the agents into a cyclic, stateful courtroom simulation.
*   **Tasks:**
    1.  Define the `CourtroomState` schema (transcript, current turn, active objections).
    2.  Build the LangGraph node network:
        *   `User Turn` -> `AI Opponent Turn` -> `AI Judge Turn`.
    3.  Implement the **"Objection!" Interrupt System**: Allow agents to read the current speaker's transcript and interrupt the graph if an objection is detected.
    4.  Implement state persistence (saving the trial progress to a database so users can resume later).
*   **Deliverable:** A fully functioning LangGraph execution loop representing a mock trial.

## Phase 5: API Layer & Frontend Integration
*   **Goal:** Expose the orchestration engine to the frontend team.
*   **Tasks:**
    1.  Create WebSocket endpoints for real-time streaming of the trial transcript.
    2.  Create REST endpoints for the **"Suggestion Engine"** (Co-Pilot mode) for users when they get stuck.
    3.  Create endpoints to generate the final **Performance Report** based on the trial's outcome.
*   **Deliverable:** API documentation (Swagger/OpenAPI) and endpoints ready for the frontend team to consume.

## 6. Testing & Evaluation Strategy

How do we ensure the AI agents are acting like real Indian lawyers and a fair judge, without hallucinating?

### Strategy 1: The "LLM-as-a-Judge" Evaluator
We will use a superior model (e.g., Gemini 1.5 Pro) strictly as an evaluator. During CI/CD or testing phases, it will read transcripts generated by the agents and score them on:
1.  **Legal Accuracy:** Did the agent cite real Indian laws (IPC, CrPC, BNS, BSA)? (0-10)
2.  **Role Adherence:** Was the prosecutor aggressive? Was the judge impartial? (0-10)
3.  **Hallucination Check:** Did the agent invent evidence that wasn't in the RAG context? (Pass/Fail)

### Strategy 2: "Mock Trial" Automated Tests
We will build automated scripts that run the LangGraph simulation without a human user.
*   **Setup:** AI Prosecutor vs. AI Defender.
*   **Execution:** Let them argue for 10 turns on a known case (e.g., a standard theft case).
*   **Assertion:** Verify that the Judge correctly stops the trial, rules on objections, and outputs a coherent final verdict.

### Strategy 3: Objection Validity Testing
*   **Unit Tests for Objections:** Feed the agents statements that are clearly "Hearsay" or "Leading the Witness."
*   **Assertion:** Verify that the opposing agent catches the fallacy and raises an objection, and verify that the Judge sustains it based on the Indian Evidence Act (or Bharatiya Sakshya Adhiniyam).

# AI Courtroom
- We want to build ai powered gamified courtroom. In which law student, practioner, lawyer can practice laws here on real cases or they can upload their cases of client.
- We will have three main entity: Judge, Complainant's Counsel/Prosecutor and Defender
- Visualise this: we will have on our frontend side like this: First it will be asked from user whether they want to practice with Complainant's Counsel/prosecutor or defender side. after this they will asked to choose cases(pre-made) or they can upload their cases details. Then gamified courtroom will appear. judge sitting in center, then prosecutor and defender and some audience sitting. 
- In left side there will be chat going on like evey thing discussion happen appear in this chat window. 
- My team is working on frontend. You and I need to focus only on backend and ai agents.
- So I want to make three ai agents for each of them(Judge, Counsel/Prosecutor and Defender). Each of them will have at least same context(Case details, evidences, whole chat window of all of them -> here we can use rag). all agent will work individually. and they will have specialised role in itself. 
- Like judge will have capability to search old cases, specialised in unbiased answer, web search, listed and understand both side view. 
- Complainant's Counsel/prosecutor will have to help his client to win the case if it is victim then try to support with evidence and social justice and all thing that help me to win.
- Defender side will have expertise in countering counsel/prosecutor. Give strong argument with real evidence or facts based on web search.
- See when lets say user(practicioner) want to play using either any side we will give suggestion or what he selected i mean,in right side chat windows will be going on. when user turn will come there will be button to execute its agent because here our goal is to make him learn and he did not finding enough counter then he can see the ai suggestion.
- tech stack: uv, bun(if needed), python, RAG, fast api, Langchain, langgraph, etc
- We ony focus on backend and we will give api end points to frontend people
- I have gemini api, sarvam api, perplexity api for web search
- We will focus on Indian Law only.
- At last we will give a report to user where he does well and where he need to improve. 
- In between they can have objection belord, objection over ruled or sustain kind of thing. we want to make it real. 
- For that we need to run all agents parallely and observing each other argument. 
- For now we will rely LLM on capability to answer and argue and knowlege mind. For orchestration task we can use lang graph, lang chain. Then later if we will have time we can fine llm for this.
